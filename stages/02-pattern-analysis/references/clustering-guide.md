# Clustering Guide

## How to Identify Clusters

Look for ideas that share any of these connectors:

1. **Same domain** — Multiple ideas in the same subject area (e.g., several podcast-related requests)
2. **Same problem shape** — Different domains but structurally similar problems (e.g., "automate X reporting" across different X's)
3. **Same audience** — Ideas serving the same type of person (e.g., solo creators vs. enterprise teams)
4. **Same deliverable type** — Ideas that would produce the same kind of output (e.g., several that are all "workflow automation" requests)
5. **Dependency chains** — Ideas where one enables another (e.g., "research tool" enables "content production pipeline")

## Naming Clusters

A good theme name is:
- A noun phrase, 2-5 words
- Describes the shared need, not the individual ideas
- Would make sense to someone who hasn't seen the individual cards

Good: "Content Production Automation", "Research-to-Output Pipelines", "Community Knowledge Management"
Bad: "Miscellaneous requests", "Good ideas", "Stuff about AI"

## Cluster Size Guidelines

- **2-4 ideas**: Tight cluster. Likely a specific, actionable need.
- **5-8 ideas**: Strong signal. This theme has real demand.
- **9+ ideas**: Probably too broad. Look for sub-clusters within the group.

If a cluster grows beyond 8, ask: "Are there two distinct needs hiding in here?" Split if yes.

## Orphan Ideas

Ideas that genuinely don't fit any cluster aren't less valuable — they may be novel. Mark them as orphans but don't discard them. In Stage 04, orphans get evaluated individually. A high-impact orphan can outrank an entire cluster.

## Cross-Cluster Connections

Some ideas bridge two themes. Don't force them into one. Instead:
- Place the idea in the cluster where it fits best
- Add a `**Cross-link:**` note pointing to the other cluster
- In the pattern map, list cross-links as a separate section

These bridges often reveal opportunities for combined deliverables that serve multiple needs at once.
