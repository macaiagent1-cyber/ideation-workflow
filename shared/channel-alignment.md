# Channel Alignment

## Content Pillars

| Pillar | Description | Audience Fit |
|--------|-------------|--------------|
| MWP workspace builds | End-to-end builds of folder-based agent workflows for real use cases | Directly solves the audience's desire for practical, immediately usable deliverables |
| Context engineering | Techniques for structuring prompts, context files, and instructions that guide AI agents effectively | Bridges the gap between knowing how to code and knowing how to orchestrate AI agents |
| AI workflow design | Architectural patterns for multi-step AI pipelines without heavyweight frameworks | Gives developers familiar mental models for building agent systems |
| Tool reviews | Evaluations of AI tools, models, and platforms through the lens of practical workflow integration | Saves the audience research time and surfaces tools they can adopt immediately |
| Community showcases | Highlighting community-built workspaces, adaptations, and domain-specific implementations | Shows real-world proof that MWP works across domains, motivating new builders to start |

## Target Audience

- **Primary audience:** AI-curious developers and technical creators who know programming but are new to agent orchestration
- **What they care about:** Practical tools over theory
- **What they already know:** Programming and software development fundamentals, but not agent orchestration or AI workflow design
- **What makes them engage:** They engage when they see something they can use immediately

## Strategic Goals

- **Channel mission:** Teaches developers how to build AI agent workflows using folder structure instead of frameworks
- **Research thesis:** The MWP/ICM methodology — folder structure as agent architecture — replaces framework-level orchestration for sequential, human-reviewed workflows
- **Community value:** Delivering working MWP workspaces to community members solves their problems AND demonstrates the thesis
- **Content flywheel:** Every workspace built generates content (build process → demo → use-case story → research validation)

## Alignment Test

When evaluating whether an idea aligns with the channel, ask:

1. **Does delivering this generate content?** — The build process, the demo, or the community response should be filmable or writable
2. **Does it reinforce the thesis?** — Ideally it demonstrates that MWP works for a new domain. If not MWP-fit, does it at least relate to AI workflows, context engineering, or agent architecture?
3. **Does the audience care?** — Would AI-curious developers and technical creators who know programming but are new to agent orchestration watch a video about building/using this?
4. **Is it shareable?** — Would someone who received this workspace share it or recommend it?
